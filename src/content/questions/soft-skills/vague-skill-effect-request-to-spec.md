## 参考答案

### 🎬 场景描述

面试官说：「你是项目的 TA，负责技能特效的技术方案。某天策划给你提了一个需求文档，上面只有一句话：

> 「角色释放大招时，特效要**看起来很酷**，要有**震撼感**。」

美术那边也没有明确的参考，只说「看着办，做出来看看」。

这种场景在实际项目中非常常见。你会怎么处理？怎么把这句话变成团队可以执行的技术方案？」

### ✅ 核心要点

1. **模糊需求≠无效需求**：策划说"很酷"不是在偷懒，是他真的还没想清楚——你的工作是用专业能力帮他想清楚
2. **结构化翻译**：把"很酷"拆解为可视化的维度（冲击力 / 节奏感 / 空间感 / 程序化细节）
3. **参考驱动对齐**：用参考视频 / GIF 建立共同语言，比文字描述高效 10 倍
4. **分层技术方案**：将特效拆为 Base Layer + Impact Layer + Environmental Layer，每层独立可调
5. **快速原型迭代**：先做"丑但可玩"的灰盒原型，让策划玩到 → 反馈 → 迭代，而不是做完美 Demo
6. **预期管理**：在方案中明确标注「性能预算」和「风险项」，防止后期返工

### 📖 深度展开

#### 解决思路（从"很酷"倒推技术方案）

```
策划说："很酷、有震撼感"
  ↓
Step 1：翻译"很酷"→ 具体视觉维度
  ├── 冲击力：瞬间画面变化（闪屏 / 震屏 / 顿帧）
  ├── 空间感：改变场景氛围（暗化 / 色调偏移 / 体积光）
  ├── 节奏感：时间结构（蓄力 → 爆发 → 余韵，不是匀速）
  ├── 程序化细节：粒子 / 拖尾 / 地面裂纹 / 环境互动
  └── 角色表演：动作设计配合（Pose 感 / 慢动作 / 特写镜头）
  ↓
Step 2：收集参考（最关键的一步）
  ├── 找 2-3 个类似游戏的大招参考（如《原神》角色 Q、《永劫》大招）
  ├── 标注每个参考中"酷"在哪个瞬间（0.5s 的闪白？地面扩散波？）
  └── 让策划指出"我要的就是这种感觉"
  ↓
Step 3：把参考翻译为技术分层
  ├── Layer 0：角色表演（动画 + 慢动作 + Camera 特写）→ 动画组负责
  ├── Layer 1：冲击瞬间（闪屏 + 震屏 + Hit Stop 顿帧）→ 特效 + 程序
  ├── Layer 2：核心特效（爆发粒子 + 冲击波 + 地面裂纹）→ 特效 + TA(Shader)
  ├── Layer 3：环境反应（暗化背景 + 体积光 + 环境粒子）→ TA(Renderer Feature)
  └── Layer 4：UI 配合（技能名弹出 + 黑边 + 时间减速）→ UI + 程序
  ↓
Step 4：性能预算分配
  ├── 总预算：GPU +8ms / CPU +3ms / 内存 +30MB
  ├── 按层分配 → 标注哪层可砍 → 让策划做选择题
  ↓
Step 5：原型制作（灰盒阶段）
  ├── 用引擎自带粒子 + 纯色 Shader 搭出节奏感
  ├── 不追求好看，追求"时间和节奏对了"
  └── 让策划玩了之后说"太快了/太慢了/冲击不够"
```

#### 知识点拆解（倒推树）

```
"很酷"→ 可执行方案
├── 需求翻译能力
│   ├── 视觉维度拆解（冲击力 / 节奏 / 空间感 / 细节密度）
│   ├── 情绪曲线映射（蓄力 → 爆发 → 余韵 → 归零）
│   ├── 参考收集与标注（视频片段 + 时间戳 + 关键帧截图）
│   └── 竞品对标分析（同类游戏大招特效拆解）
├── 技术分层设计
│   ├── Layer 0: 角色表演（Animation + Hit Stop + Camera）
│   ├── Layer 1: 冲击反馈（Screen Flash + Camera Shake +顿帧）
│   ├── Layer 2: 核心特效（Particle System + Shader + Trail）
│   ├── Layer 3: 环境反应（Post-Process + 体积雾 + 环境粒子）
│   ├── Layer 4: UI 配合（技能名演出 + 黑边 + 慢动作）
│   └── 时间轴编排（每层的开始/峰值/结束时间点）
├── 性能预算管理
│   ├── 总预算评估（基于目标设备帧率）
│   ├── 按层分配 + 可砍项标注（让策划做选择题不是判断题）
│   ├── LOD 策略（高端机全特效，低端机砍 Layer 3-4）
│   └── 风险项标注（如：Compute Shader 老设备不支持）
├── 原型迭代流程
│   ├── 灰盒原型（纯几何体 + 基础粒子，验证节奏感）
│   ├── 半成品原型（加入 Shader 和关键特效，验证视觉方向）
│   ├── 润色阶段（细节打磨、参数微调、音效同步）
│   └── 验收标准（每个维度有明确的 Pass/Fail 条件）
└── 沟通策略
    ├── 用视频而不是文字对齐（共享一个参考视频 = 减少会议 50%）
    ├── 让策划做选择题不做问答题（"你要 A 风格还是 B 风格？"）
    ├── 原型验收用录屏发给远程成员
    └── 里程碑节点同步（不要做完才给看）
```

#### 特效时间轴 Spec 模板

```
大招特效时间轴（总时长 3.5s）

时间    │ Layer 0 (角色)   │ Layer 1 (冲击)   │ Layer 2 (特效)   │ Layer 3 (环境)
────────┼──────────────────┼──────────────────┼──────────────────┼──────────────────
0.0s    │ 蓄力 Pose 开始   │                  │                  │ 背景开始暗化
0.5s    │ 蓄力到极点       │                  │ 脚下能量聚集     │ 暗化到 60%
0.8s    │ 爆发 Pose!       │ 闪屏白(0.1s)     │ 冲击波扩散       │ 全屏暗化→恢复
1.0s    │ 慢动作 0.3x      │ 震屏(强度0.8)    │ 地面裂纹蔓延     │ 体积光柱出现
1.5s    │ 恢复正常速度     │ 震屏衰减         │ 主特效全开       │ 环境粒子飘散
2.5s    │ 归位 Pose        │                  │ 特效收束消散     │ 暗化完全恢复
3.5s    │ 技能结束         │                  │                  │

性能预算：GPU +6ms / CPU +2ms / 内存 +20MB
可砍层：Layer 3（环境）→ 省 3ms GPU / Layer 4（UI）→ 省 1ms CPU
```

**模糊→结构化翻译表：**

| 策划说的 | 翻译为 | 技术方案 |
|---------|--------|---------|
| "很酷" | 冲击力强 | Hit Stop 顿帧 + 闪屏 + 震屏 |
| "有震撼感" | 空间感被改变 | 全屏暗化 + 体积光 + 环境粒子 |
| "要帅气" | 角色姿势好看 | 慢动作 + Camera 特写角度 |
| "要爽" | 节奏感对 | 蓄力→爆发→余韵的时间曲线 |
| "再来点感觉" | 细节密度不够 | 加拖尾、加裂纹、加环境互动 |
| "太花了" | 信息过载 | 减少同屏粒子数 / 降低闪光频率 |
| "不够霸气" | 规模感不够 | 增大冲击波范围 / 加全屏效果 |

**原型迭代流程（推荐 3 轮）：**

```
Round 1：灰盒节奏原型（1天）
  ├── 目标：验证时间轴和节奏感
  ├── 产出：纯色几何体 + 基础粒子 + 震屏/闪屏参数
  └── 验收：策划说"节奏对不对？"→ 调整时间轴

Round 2：视觉方向原型（2-3天）
  ├── 目标：确定美术风格和特效方向
  ├── 产出：加入 Shader（冲击波、地面裂纹）、关键粒子替换
  └── 验收：策划说"这个方向对不对？"→ 调整视觉风格

Round 3：润色定版（3-5天）
  ├── 目标：细节打磨 + 性能优化 + 音效同步
  ├── 产出：完整特效 + LOD 分级 + 性能报告
  └── 验收：策划+美术+程序三方评审 → 定版
```

### ⚡ 实战经验

- **最大的坑是跳过原型直接做成品**：特效做了两周，策划看完说"不是我要的感觉"——两周白费。灰盒原型 1 天就能避免这个风险
- **参考视频 > 文字描述 > 口头沟通**：给策划看 3 个参考视频，让他指出"我要这个 0:15 秒的那种感觉"，比开 1 小时会有效
- **让策划做选择题**：「方案 A 冲击力更强但贵 3ms，方案 B 稳定但视觉平淡，你选哪个？」而不是「你想要什么效果？」
- **性能预算前置沟通**：在方案阶段就把性能预算告诉策划，防止他爱上一个做不出来的效果
- **特效 != 粒子**：很多 TA 只想着粒子系统，但"震撼感"更多来自时间控制（顿帧、慢动作）和空间控制（暗化、Camera 运动），这些不需要 GPU 算力
- **记录"为什么"**：每个设计决策记录原因（"闪屏用白色而非角色色，因为策划参考的是《鬼泣》风格"），后续改需求时可追溯

### 🎯 能力体检清单

| 卡点 | 盲区 | 补习方向 |
|------|------|----------|
| 收到"很酷"后直接动手做 | 需求翻译意识 | 学习结构化需求拆解方法 |
| 做完给策划看，被打回 | 缺少原型迭代流程 | 学习灰盒原型 / 敏捷特效开发 |
| 不知道怎么找参考 | 竞品分析方法 | 练习拆解 AAA 游戏特效时间轴 |
| 性能超预算被砍 | 预算前置管理 | 学习 Performance Budget 制定 |
| 团队各做各的，合不起来 | 跨组协作编排 | 学习特效时间轴 / 动画事件同步 |
| 说不清自己方案"为什么酷" | 视觉冲击力原理 | 学习视觉心理学（对比/节奏/规模） |

## 参考答案

### 🎬 场景描述

> 面试官：假设这样一个场景——美术总监说角色渲染效果"不够通透"（原话），要求改。程序看了Shader说"计算没问题，PBR流程是对的"。两边僵持了一周，项目进度受影响。作为TA你会怎么处理？
>
> 追问：如果美术的要求在当前性能预算内确实做不到呢？你会怎么沟通？

### ✅ 核心要点

1. **TA的核心定位是"翻译官"**：不是画画的也不是写代码的，是把"不好看"翻译成"哪个参数不对"的人
2. **结构化沟通框架**：先对齐问题定义（什么是"通透"），再拆解技术原因，最后给出可选方案和代价
3. **管理预期是关键能力**：在性能预算内给出"最好能做到的效果"，而不是直接说"做不到"
4. **建立参考物沟通**：用截图标注、Side-by-side对比、A/B切换代替口头描述
5. **文档化决策**：每次效果对齐的结论要形成文档，避免同样的讨论重复发生

### 📖 深度展开

#### 解决思路（从效果倒推实现）

```
"不够通透" ← 这是模糊的感性描述
  ↓ Step 1: 把感性描述翻译为可量化指标
"通透"可能 = 皮肤SSS不够？环境光太暗？高光太锐利？色彩饱和度不够？
  ↓ Step 2: 制作对比参照
Side-by-side截图：当前效果 vs 目标参考图，标注差异区域
  ↓ Step 3: 拆解技术方案
每个差异 → 对应的Shader参数/光照设置/后处理调整
  ↓ Step 4: 给出方案矩阵（效果 vs 性能代价）
让美术总监和主程做选择题，而不是判断题
```

```
TA跨部门沟通

│   │   ├── "厚重" → 饱和度 / 对比度 / 环境光遮蔽
│   │   ├── "糊" → Bloom强度 / 景深范围 / 分辨率
│   │   ├── "塑料感" → Roughness范围 / 高光强度 / F0值
│   │   └── "飘" → 锚点权重 / 质心 / 动画曲线

│   │   └── 逐要素对比当前效果
│   └── A/B快速验证

├── 方案沟通层（让非技术人员做选择题）
│   ├── 方案矩阵法

│   ├── 成本可视化
│   │   ├── 帧时间预算饼图
│   │   └── "花1.2ms买30%效果提升值得吗？"

    │   ├── 每类材质的参数范围
```

#### 沟通方案对比表

| 沟通方式 | 场景 | 效果 | TA建议 |
|---|---|---|---|
| 口头传达 | "美术说改通透点" | ❌ 程序不懂"通透"是什么操作 | 永远不要纯口头传达渲染需求 |
| 文字描述 | "SSS强度调高，高光范围调大" | ⚠️ 美术不懂SSS是什么 | 有进步但美术仍然无法判断 |
| 截图标注 | 在截图上圈出区域+写参数 | ✅ 双方能看到同一个东西 | 推荐的日常沟通方式 |
| 实时调参 | 引擎内挂参数面板，美术实时拖 | ✅✅ 最高效 | 最终对齐阶段必用 |
| 方案矩阵 | A/B/C方案+效果+性能代价 | ✅✅ 决策级沟通 | 正式评审时使用 |

**实战对话还原：**

```
【错误示范】
美术：角色不够通透，能不能改？
程序：PBR流程没问题啊，Shader计算是对的。
美术：（不满意但说不出原因）反正就是不对……
程序：（防御性）那你说哪里不对？
→ 僵局

【正确示范（TA介入）】
TA：我看了下当前效果，截了几张对比图。
[展示 Side-by-side：当前效果 vs 目标参考]
TA：我理解美术说的"通透"主要是指皮肤看起来偏黄偏暗，
    对比参考图，差异可能在这三个点：
    1. 皮肤SSS的半径值偏低（当前0.15，参考图大概0.25的效果）
    2. 阴影区域的AO太重（AO强度0.8可能偏高）
    3. 高光太锐利（Roughness范围偏窄）
美术：对对对！就是那个感觉！特别是鼻子和脸颊那里！
TA：好的。我做了三个方案：
    方案A：调SSS半径 0.15→0.25，效果提升明显，GPU +0.8ms
    方案B：调AO强度 + Tone Mapping曲线，效果中等，GPU +0ms  
    方案C：A+B都做，效果最好，GPU +0.8ms
主程：当前GPU预算还剩1.2ms，方案C可以接受。
美术：那就C！
TA：好的，我先用方案C做一版，明天Review时实机看效果。
→ 问题解决，各方满意
```

### ⚡ 实战经验

1. **永远带"参照物"沟通**：口头描述渲染效果是灾难。养成习惯：任何效果讨论都带截图、参考图、对比标注。一张图的信息量抵得上30分钟会议
2. **"做不到"是最差的回答**：如果真的性能不够，正确的说法是"在当前0.5ms预算内，我能做到这个效果（展示Prototype），如果要做到你说的100%，需要额外1ms，你看要不要跟主程申请预算"。把选择题留给决策者
3. **建立"美术词汇→技术参数"翻译表**：每个项目的美术团队都有自己的黑话。花两周记录"美术说的词"→"对应的参数/Shader改动"，形成团队翻译词典。这个文档价值千金
4. **TA是桥梁不是裁判**：不要站在程序一边说"美术的需求不合理"，也不要站在美术一边说"程序写错了"。TA的立场是"我理解双方的需求和约束，这是可行的方案"
